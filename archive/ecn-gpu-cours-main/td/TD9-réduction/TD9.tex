  \section{Produit scalaire}
%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{objectif}[]{}
 \begin{itemize}
	\item observer l'influence des warps
	\item mettre en {\oe}uvre un algorithme d'arbre de réduction
\end{itemize}
\end{objectif}

\begin{wtodo}[Produit scalaire]{}

Implémentez un {\it kernel} qui effectue le produit scalaire $s$ (dot en anglais) pour deux vecteurs $a$ et $b$, $s = \sum_{i = 1..n} a_i \times b_i$.\\

Pour vous aider, utilisez le squelette fourni dans le fichier {\tt dot.cu} qui charge le tableau {\tt cache} en mémoire partagée avec le produit de $a$ et $b$ (les conditions aux limites sont prises en charge).\\

Votre travail consiste donc à simplement faire la somme (la réduction) des éléments qui sont dans {\tt cache}.
Les sommes réalisées par les blocks seront additionnées par le CPU (voir fin du code dans le fichier {\tt dot.cu}). Le retour du calcul dans un block se fait via la variable {\tt c[blockIdx.x]}.\\


% Cependant, pour des raisons de simplicité, vous pouvez supposer que les vecteurs d'entrée comptera au maximum 2048 x 10240 éléments afin qu'ils puissent être traitée par un seul lancement de kernel. 

Réalisez deux versions du calcul, l'une sans prendre en compte les warp et l'autre avec. Comparez les performances (avec gprof et nvprof) en modifiant la taille des données (par exemple avec des vecteurs de taille 2048 x 10240)

\end{wtodo}
